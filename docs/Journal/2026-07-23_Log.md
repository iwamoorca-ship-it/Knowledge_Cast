# Architecture Review

当初の構成

```
News

↓

Summary

↓

Script

↓

TTS

↓

MP3

↓

feed.xml
```

を見直した。

議論の結果、

PublisherはMP3ではなく
Episode Metadataを入力とすべきとの結論に至る。

改善後

```
Episode JSON

↓

build_feed.py

↓

feed.xml
```

これにより、

RSS Generatorは
Knowledge Generationから完全に独立した。

---

# Design Decision

Episode MetadataをArtifactとして扱う。

Publisherは

- AI
- News
- TTS

を一切知らない。

必要なのは

```
episode.json
```

のみ。

これにより、

Podcast

Web

Notion

Mail

など複数Publisherへの展開が可能となる。

---

# Specification

- Episode JSON Specification

```
Episode JSON

feed.xml
```

Git操作は含めない。

# Publisher Design

Publisher層を独立させる設計へ変更。

役割

```
build_feed.py

↓

publish.py

↓

GitHub Push
```

Publishは

Git操作のみを担当する。

# Design Philosophy

KnowledgeCastは
Podcast作成ツールではない。

Knowledge Platformにおける

Artifact Publisher

のPoCである。

Knowledge Generationと
Artifact Publicationを
完全に分離する設計とした。
